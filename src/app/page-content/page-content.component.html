Angular is a framework for building client applications.

This is a simple example of a component.

The component is the most basic building block of an Angular application.

A component controls a portion of the screen — a view — through its associated template.

A component is made up of three things:

  • A component class that handles data and functionality.
  • An HTML template that determines the UI.
  • Component-specific styles that define the look and feel.

Here's an example of a component's file structure:

    app
    ├── app.component.css
    ├── app.component.html
    ├── app.component.spec.ts
    └── app.component.ts

The CLI generated three files for the AppComponent, and a fourth file for the application routing.

The files are the following:

  • app.component.ts— the component class code, written in TypeScript.
  • app.component.html— the component template, written in HTML.
  • app.component.css— the component's private CSS styles.
  • app.component.spec.ts— the component test file, written in Jasmine.

The CLI generated a root component that is named AppComponent and displays the application title.

The component class is where you handle data and functionality for the view.

The template is where you define how the view is presented to the user. The template and styles are optional if all you're doing is generating the application shell.

The CLI generated the root AppComponent in the src/app folder. You can find the app.component.ts file there.

The CLI also generated app.module.ts, which is the root module of the application. It tells Angular how to assemble the application. Initially app.module.ts declares only the AppComponent. As you add more components to the application, they must be declared here.

The app.component.ts file contains the following import statements that import the Component symbol from the Angular core library and the environment object from the environments folder. 

    import { Component } from '@angular/core';
    import { environment } from '../environments/environment';

The @Component decorator identifies AppComponent as an Angular component. The decorator provides metadata about the component, including its templates, styles, and a selector.

The CLI generated the component's selector, app-root, and the index.html file that references that selector.

The templateUrl and styleUrls properties tell Angular where to find the component's template and style files.

The component definition also exports the class, AppComponent, so that you can use it in other components' templates.

The AppComponent class contains a title property. The value of this property is displayed in the application by using the interpolation syntax in the index.html template.
